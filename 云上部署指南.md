# 智能充电桩推荐系统云上部署指南（Docker Compose）

本文档基于当前仓库结构，目标是把后端 + 管理端 + MySQL + Redis 一次性部署到云服务器。

## 1. 推荐云资源

- 系统：CentOS 7/8/9（推荐 2 核 4G 起步）
- 磁盘：40G 以上
- 开放端口：`22`（SSH）、`80`（HTTP）
- 可选：`443`（HTTPS）

## 2. 首次初始化服务器

```bash
sudo yum install -y yum-utils device-mapper-persistent-data lvm2 git curl
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl enable --now docker
sudo usermod -aG docker $USER
newgrp docker
docker --version
docker compose version
```

CentOS 常见网络与安全策略建议：

```bash
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

# 如遇到容器网络访问限制，可临时放宽 SELinux（生产建议按最小权限单独配置策略）
sudo setenforce 0
```

## 3. 拉取代码

```bash
git clone <你的仓库地址> charging-recommendation-system
cd charging-recommendation-system
```

## 4. 配置生产环境变量

```bash
cd deploy
cp .env.example .env
```

修改 `deploy/.env`，至少替换：

- `MYSQL_ROOT_PASSWORD`：改为强密码

## 5. 启动服务

在项目根目录执行：

```bash
docker compose -f deploy/docker-compose.yml --env-file deploy/.env up -d --build
```

查看状态：

```bash
docker compose -f deploy/docker-compose.yml ps
```

查看日志：

```bash
docker compose -f deploy/docker-compose.yml logs -f backend
```

## 6. 访问方式

- 管理端：`http://云服务器公网IP/`
- 后端接口（通过 Nginx 反代）：`http://云服务器公网IP/api/...`

说明：

- 管理端容器内置 Nginx，已配置把 `/api/` 转发到后端 `backend:8081`。
- 前端管理端代码中的 `baseURL: '/api/v1/admin'` 可直接工作。

## 7. 小程序（frontend-app）联调说明

当前 `frontend-app/app.js` 中 `baseUrl` 写死为局域网 IP。部署到云后，需要改成公网地址，例如：

```js
globalData: {
  baseUrl: 'https://api.your-domain.com'
}
```

如果暂时不用独立 API 域名，也可以先用：

```js
baseUrl: 'http://云服务器公网IP'
```

然后在小程序后台把服务器域名加入合法域名白名单。

## 8. HTTPS（建议）

如果你有域名，建议使用 Nginx + Let’s Encrypt：

1. 域名 A 记录指向云服务器公网 IP。
2. 在 CentOS 服务器安装 certbot。
3. 申请证书并把管理端 Nginx 配置升级为 443。

也可以接入云厂商负载均衡（SLB/CLB）直接托管证书。

## 9. 常用运维命令

重启：

```bash
docker compose -f deploy/docker-compose.yml --env-file deploy/.env restart
```

停止：

```bash
docker compose -f deploy/docker-compose.yml --env-file deploy/.env down
```

更新代码后重新发布：

```bash
git pull
docker compose -f deploy/docker-compose.yml --env-file deploy/.env up -d --build
```

## 10. 故障排查

- 后端连不上 MySQL：检查 `deploy/.env` 中密码是否与 MySQL 容器一致。
- 页面打开空白：看 `admin` 容器日志与浏览器控制台。
- `/api` 404：确认请求路径是否以 `/api/` 开头。
- 小程序请求失败：检查小程序合法域名配置和 `baseUrl`。
